# Phase 12 - Phase 7.2 Core Chat Experience and Knowledge Grounding

## Objective
Complete Phase 7.2 by delivering a production-ready core chatbot experience with retrieval-grounded responses, source attribution, confidence signaling, feedback capture, and role-safe context boundaries.

## Scope Covered
- Chat UX completion for authenticated users:
  - asynchronous ask flow with anti-forgery protection
  - typing indicator, retry path, and user-visible error states
  - conversation history refresh with assistant feedback actions
- Retrieval-grounded response path:
  - approved content retrieval from CMS pages/notices and support settings
  - appointment and billing workflow guidance as explicit knowledge sources
  - patient-scoped summary context for own appointments/bills only
- Source-aware response shaping:
  - confidence score and source list returned with each answer
  - source citation line enforced in responses
- Security and privacy boundaries:
  - session ownership checks enforced for history and feedback submission
  - no cross-user patient data context exposure in retrieval logic

## Key Technical Changes
1. Knowledge retrieval and grounding service:
- Added:
  - `Services/Implementations/ChatbotKnowledgeService.cs`
  - `IChatbotKnowledgeService` contract in `Services/Interfaces/IServices.cs`
- Grounded sources include:
  - published CMS pages
  - active CMS notices
  - support contact setting
  - role-aware module guidance for appointment/billing
  - patient-specific summaries scoped by authenticated user id

2. Chat orchestration enhancement:
- Updated `OpenAiChatbotService` to:
  - fetch grounded context before provider call
  - pass context into prompt builder
  - attach `Sources` and `ConfidenceScore` in `ChatbotAskResponse`
  - suggest escalation when confidence is low
  - persist feedback through `SubmitFeedbackAsync`

3. Prompt and model contract updates:
- Updated `IChatbotPromptBuilder` and `ChatbotPromptBuilder` to include grounded context and source-citation instruction.
- Expanded chatbot response/domain models in `Models/Chatbot.cs`:
  - `ChatKnowledgeContext`
  - `ChatKnowledgeSource`
  - `ConfidenceScore` and `Sources` on `ChatbotAskResponse`

4. Controller/API and UI workflow:
- Updated `ChatbotController`:
  - added `AskJson` endpoint for asynchronous ask flow
  - added `SubmitFeedback` endpoint
  - retained server-render fallback path
- Updated `Views/Chatbot/Index.cshtml`:
  - AJAX-ready form, retry action, typing/error states
  - source attribution card and confidence display
  - feedback buttons on assistant messages
- Added frontend behavior script:
  - `wwwroot/js/chatbot-ui.js`

5. Dependency injection wiring:
- Registered `IChatbotKnowledgeService` in `Program.cs`.

6. Test coverage for 7.2 behavior:
- Added `tests/MedyxHMS.Tests/Services/ChatbotKnowledgeServiceTests.cs`
- Coverage includes:
  - CMS/knowledge source matching
  - patient data authorization scoping
  - fallback/support source presence

## Risks and Follow-up
- Current retrieval is keyword-based and intentionally bounded to approved sources; semantic/vector retrieval can be introduced in a later sub-step.
- Admin runtime controls for chatbot role toggles and analytics remain for Step 7.3.
- Additional abuse controls (rate limits per route/user) should be layered in Phase 7.4 hardening.
