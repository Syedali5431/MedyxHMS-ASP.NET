# PH17 - Public Site Content Studio and SuperAdmin Dashboard Section Governance

## Objective
Add controlled content-management capabilities so only Admin and SuperAdmin users can manage public-facing pages and page presentation, while keeping dashboard section/module enablement restricted to SuperAdmin.

## Scope Delivered
- Extended CMS page editing with image upload and typography controls.
- Added Home, Contact, and Location page customization in public-site settings.
- Enabled automatic publication of newly created CMS pages to the public main-site navigation (not dashboard lists) when published and menu-enabled.
- Enforced SuperAdmin-only module/dashboard section governance paths.

## Key Implementation Changes
- CMS page model/viewmodel now supports:
  - featured image path
  - font family
  - font size
- CMS create/edit flow now:
  - accepts image upload
  - stores style metadata
  - auto-syncs CMS pages to `CmsMenuItems` when `Published + ShowInMenu`
  - removes linked menu items when a page is deleted
- Public Site Admin settings now manage:
  - Home title, tagline, description, font, hero image
  - Contact description, font, hero image
  - Location description, font, hero image
- Public views updated to render configured image/style/text values.
- Module management user-level access endpoints reduced to SuperAdmin only.

## Security and Access Model
- Public content editing remains guarded by role authorization:
  - `PublicSiteAdminController`: `Admin,SuperAdmin`
  - `CmsController`: admin policy (`RequireAdminRole`)
- Dashboard section/module enablement controls are now SuperAdmin-only in `ModuleManagementController`.

## Validation Notes
- Compile validation is required after introducing new viewmodel fields, controller dependencies, and model properties.
- Existing databases are supported through bootstrap SQL upgrades for new CMS columns.

## Operational Impact
- Admin/SuperAdmin can now maintain core marketing/public pages without code changes.
- Newly added public pages are visible to website visitors through the public navigation once published.
- Dashboard section/module authority is now centralized to SuperAdmin.
