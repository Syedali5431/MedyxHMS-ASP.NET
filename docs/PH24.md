# PH24 - Mandatory Unique User Name and Numeric User ID Policy

## Objective
Enforce a consistent account identity policy:
1. `user_name` is mandatory
2. `user_name` is unique
3. User IDs are numeric values (`1`, `2`, `3`, ...)

## Scope Delivered
- Added mandatory `UserName` input and validation to:
  - account signup model (`RegisterViewModel`)
  - staff creation DTO (`StaffCreateDto`)
  - patient portal register DTO (`PatientPortalRegisterDto`)
- Added UI fields for `UserName` in:
  - account signup view
  - staff create view
- Enforced duplicate checks against normalized username in:
  - account self-signup flow
  - admin/superadmin staff creation flow
  - patient portal registration flow
  - staff and patient service-level create paths (defensive checks)

## Numeric User ID Behavior
- New users created through account signup, staff creation, and patient portal registration now receive the next numeric string ID based on max existing numeric user ID.
- SuperAdmin seed identity updated to numeric ID `1` for fresh provisioning paths.

## Data and Schema Safeguards
- EF configuration enforces required and unique username behavior using indexes on `ApplicationUser`.
- Runtime database initializer now:
  - backfills null usernames
  - normalizes missing normalized usernames
  - resolves duplicate usernames with deterministic suffixing
  - applies NOT NULL + unique username constraints/indexes

## SQL Script Updates
- Updated `scripts/New-Database.sql` and `scripts/New-Database-Empty.sql`:
  - `AspNetUsers.UserName` and `AspNetUsers.NormalizedUserName` set to `NOT NULL`
  - added unique username indexes
  - added composite uniqueness index for `(Id, UserName)`
  - default SuperAdmin seed switched from `superadmin-user-id` to numeric `1`

## Validation
- Build and test suite executed after implementation.
- No diagnostics errors in modified files.
