# Phase 10 - Phase 6 Licensing & Commercial Control Delivery

## Objective
Add a secure, auditable licensing system that enforces expiry for staff and admin workflows while preserving SuperAdmin and patient-facing access, with renewal and reminder operations handled entirely on the server.

## Scope Covered
- Phase 6.1:
  - Added the license domain model, audit history, reminder history, and startup seed/settings support.
  - Centralized server-side status evaluation and role-aware restriction logic.
- Phase 6.2:
  - Added SuperAdmin-only license management UI and renewal workflow.
  - Added expired-license information screen and middleware-based enforcement.
- Phase 6.3:
  - Added 5-day reminder evaluation, configurable reminder templates, manual resend support, and daily background execution.
- Phase 6.4:
  - Added licensing test coverage for core renewal, reminder, and access-control rules.

## Key Technical Changes
1. License domain and persistence:
- Added `LicenseRecord`, `LicenseAuditLog`, and `LicenseReminderLog`.
- Registered licensing entities in `ApplicationDbContext`.
- Added startup table creation and bootstrap seed logic in `DatabaseInitializer`.
- Seeded configurable settings for reminder subject/body and contact guidance.

2. Service and enforcement layer:
- Added `ILicenseService` and `LicenseService`.
- Implemented status evaluation for `Active`, `ExpiringSoon`, and `Expired`.
- Implemented renewal validation limited to 1, 2, or 3 years.
- Implemented duplicate reminder prevention by license cycle.
- Added `LicenseEnforcementMiddleware` to redirect expired staff/admin requests.

3. Admin workflow and UI:
- Added `LicenseController`.
- Added SuperAdmin-only management screen with:
  - current expiry/status visibility
  - renewal form
  - reminder history
  - audit history
  - manual reminder resend action
- Added expired-license page with renewal contact guidance.
- Added shared layout navigation link for SuperAdmin users.

4. Background automation and tests:
- Added `LicenseReminderHostedService` for daily evaluation.
- Added `LicenseServiceTests` for:
  - status evaluation
  - invalid renewal-term rejection
  - duplicate reminder prevention
  - exemption and restriction logic

## Product Assumptions Applied
- If a license is already expired, renewal extends from the current UTC date.
- Reminder recipients are all active users with valid email addresses.
- Patient access is exempted through patient-portal route bypass and `Patient` role bypass when present.
- No grace period is applied after expiry.

## Dependencies and Risks
- SMTP configuration must be present for real reminder delivery.
- If patient identity-role seeding changes later, the current route-based patient exemption can be tightened to a pure role-based rule.
- Product rule changes for expiry grace periods or renewal-from-expiry behavior are isolated to `LicenseService` and can be adjusted without redesigning the feature.
