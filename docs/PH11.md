# Phase 11 - Phase 7.1 Chatbot Foundation and SMTP Operational Check

## Objective
Start Phase 7 implementation with a secure backend chatbot foundation and deployable SMTP operational health verification in admin notification settings.

## Scope Covered
- Phase 7.1 foundation start:
  - Implemented backend-only chatbot orchestration and safety guardrails.
  - Added initial chat session/message persistence and role-aware prompt construction.
  - Added authenticated UI entry point for chatbot interactions.
- SMTP operational readiness:
  - Added SMTP configuration and connectivity health check service.
  - Exposed health check action and status card in CMS Notification Settings.

## Key Technical Changes
1. Chatbot domain + persistence:
- Added models:
  - `ChatSession`
  - `ChatMessage`
  - `ChatFeedback`
  - moderation/result DTOs
- Registered chatbot DbSets and relationships in `ApplicationDbContext`.
- Added database bootstrap table creation for chatbot tables in `DatabaseInitializer`.

2. Chatbot services:
- Added interfaces:
  - `IChatbotService`
  - `IChatbotModerationService`
  - `IChatbotPromptBuilder`
- Implemented:
  - `OpenAiChatbotService` (backend-only provider integration with safe fallback)
  - `ChatbotModerationService` (unsafe medical and emergency guardrails)
  - `ChatbotPromptBuilder` (role-aware system prompt)
- Added OpenAI config section (`Enabled`, `BaseUrl`, `ApiKey`, `Model`) to app settings.

3. Chatbot UI and routing:
- Added `ChatbotController` and `Views/Chatbot/Index.cshtml`.
- Added navigation links in shared and patient portal layouts.
- Added chat page model `ChatbotPageViewModel`.

4. SMTP health checks:
- Added `ISmtpHealthService` and `SmtpHealthService`.
- Added SMTP health status model and CMS view model support.
- Added `RunSmtpHealthCheck` action and SMTP health panel in Notification Settings UI.
