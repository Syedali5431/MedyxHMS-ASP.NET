# PH14 - Authentication and Dashboard Routing Audit (2026-04-21)

## Objective
Validate that core ASP.NET HMS modules are reachable and authorized correctly, and ensure users are redirected to their assigned dashboard destination after successful login.

## Scope
- Controller/module coverage audit for major HMS modules.
- Login flow and post-authentication redirect validation.
- Patient portal routing path validation.
- Focused code hardening for role-aware redirection in primary account login flow.

## Findings Summary
- Major modules are present with role/authorization guards in their controllers (OPD, IPD, Billing, Lab, Prescription, BloodBank, FrontOffice, Payroll, Attendance, Leave, Reports, CMS, Chatbot, Licensing).
- Patient portal has dedicated routes and dashboard endpoint:
  - `/PatientPortal/Account/Login`
  - `/PatientPortal/Dashboard`
- Main account login previously defaulted to a generic dashboard path when no return URL was provided.

## Implemented Change
Updated primary login flow in `Controllers/AccountController.cs`:
- Replaced generic post-login redirect call with role-aware redirect handling.
- Added `RedirectToLocalAsync(ApplicationUser user, string? returnUrl)`.
- Preserved secure local return URL behavior.
- Added fallback role mapping when no return URL is supplied:
  - Patient -> `/PatientPortal/Dashboard`
  - Receptionist -> `FrontOffice/Index`
  - Accountant -> `Billing/Index`
  - Pharmacist -> `Prescription/Index`
  - Nurse -> `IPD/Index`
  - Doctor -> `OPD/Index`
  - Default -> `Dashboard/Index`

Updated project item exclusions in `MedyxHMS.csproj`:
- Added removal of `Content` items under `tests/**`.
- Added removal of `Compile`, `EmbeddedResource`, `None`, and `Content` items under `temp_build_output/**`.
- This prevents recursive copy/input expansion from generated test artifacts during main web project build.

## Validation Performed
- Confirmed redirect target controllers/endpoints exist and expose index actions.
- Confirmed patient portal dashboard route mapping exists.
- Checked editor diagnostics on updated controller; no issues reported.
- Executed `dotnet build -nologo -clp:ErrorsOnly` after project exclusion updates.
- Result: build succeeded (warnings remain).

## Risks and Notes
- Role names must match Identity role values exactly (`Patient`, `Receptionist`, `Accountant`, `Pharmacist`, `Nurse`, `Doctor`).
- Users with multiple roles will be routed by the current precedence order in code.
- If workflow policy changes, update the role precedence in `RedirectToLocalAsync`.
